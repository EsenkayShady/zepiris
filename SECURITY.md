# Security policy

## Supported versions

Security updates are applied to the **default branch** (`main` or `master`) as
practical. Use the latest tagged release or commit when deploying.

## Reporting a vulnerability

**Please do not** open a public GitHub issue for security vulnerabilities.

Instead:

1. Use **GitHub [private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability)** for this repository if it is enabled by maintainers, **or**
2. Email maintainers with a clear subject line (e.g. `[SECURITY] ZepIris`).

Include:

- Description of the issue and potential impact
- Steps to reproduce (if safe to share)
- Affected versions or components (if known)

We aim to acknowledge reports within a few business days. Critical issues may
take longer to fix and disclose responsibly.

## Scope

This policy covers the **ZepIris** application code and official Docker images
in this repository. Third-party services (Milvus, MinIO, PyTorch, etc.) follow
their own security policies.

## Safe harbor

We support good-faith security research that follows this reporting process.
